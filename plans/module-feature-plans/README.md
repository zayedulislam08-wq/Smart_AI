# Module Feature Plans

This folder contains separate planning files for each major module in the project.

## Why This Structure Is Better

- each module stays isolated and easier to update
- future feature discussions remain organized
- one module can progress without making the overall planning messy
- implementation can later follow module by module

## Recommended Update System

When a module evolves, update only its own plan file first.

Recommended workflow:

1. Update the module plan file
2. Mark which phase or feature changed
3. If priorities changed, update `ROADMAP.md`
4. Only after planning is clear, start implementation
